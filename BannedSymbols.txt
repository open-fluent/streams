T:System.DateTime;Don't use DateTime. Use NodaTime.Instant (UTC moment), NodaTime.LocalDateTime (wall clock), or NodaTime.ZonedDateTime (with zone) instead. DateTime carries no UTC-offset and DateTimeKind is a 2-bit enum that cannot model Warsaw CET/CEST (+01/+02) correctly.
P:System.DateTime.Now;Use NodaTime.SystemClock.Instance.GetCurrentInstant() or DateTimeOffset.UtcNow then Instant.FromDateTimeOffset(). DateTime.Now returns server-local time - wrong when the host is UTC (Docker/k8s) and Warsaw is UTC+1/+2.
P:System.DateTime.UtcNow;Use DateTimeOffset.UtcNow or NodaTime.SystemClock.Instance.GetCurrentInstant(). DateTime.UtcNow returns Kind=Utc DateTime that cannot round-trip through Warsaw-local APIs without NodaTime zone logic.
P:System.DateTime.Today;Use NodaTime: SystemClock.Instance.GetCurrentInstant().InZone(DateTimeZoneProviders.Tzdb["Europe/Warsaw"]).Date. DateTime.Today returns server-local midnight which skips the non-existent 02:00-03:00 gap during Warsaw DST start (last Sunday March) and is ambiguous during DST end (last Sunday October 02:00-03:00 = two distinct UTC instants).
P:System.DateTimeOffset.Now;Use DateTimeOffset.UtcNow or NodaTime.SystemClock.Instance.GetCurrentInstant(). DateTimeOffset.Now carries the server's local offset - in UTC Docker containers this is +00:00, not Warsaw +01:00/+02:00.
M:System.DateTime.SpecifyKind(System.DateTime,System.DateTimeKind);Do not use DateTime.SpecifyKind. It relabels the ticks without converting - stamping DateTimeKind.Utc on a Warsaw CEST value (14:00 +02:00) silently produces 14:00 UTC instead of 12:00 UTC. Use NodaTime: LocalDateTime.InZoneStrictly(warsawZone).ToInstant().ToDateTimeUtc() for explicit, exception-safe conversion.
M:System.DateTime.ToLocalTime;Use NodaTime: Instant.FromDateTimeUtc(dt).InZone(DateTimeZoneProviders.Tzdb["Europe/Warsaw"]). ToLocalTime() applies the server's OS timezone - correct on Warsaw-configured hosts, silently wrong in UTC containers.
M:System.DateTime.ToUniversalTime;Use NodaTime: localDateTime.InZoneStrictly(warsawZone).ToInstant().ToDateTimeUtc(). ToUniversalTime treats DateTimeKind.Unspecified as Local, silently applying server offset and corrupting Warsaw timestamps stored without Kind.
P:System.DateTimeOffset.DateTime;Do not use DateTimeOffset.DateTime. It returns Kind=Unspecified, discarding the +01:00/+02:00 Warsaw offset. Use Instant.FromDateTimeOffset(dto).InZone(warsawZone) to retain zone semantics, or .UtcDateTime for unambiguous UTC ticks.
P:System.DateTimeOffset.LocalDateTime;Do not use DateTimeOffset.LocalDateTime. It converts to the server's local timezone, not Warsaw CET/CEST. Use NodaTime: Instant.FromDateTimeOffset(dto).InZone(DateTimeZoneProviders.Tzdb["Europe/Warsaw"]).LocalDateTime.
T:System.TimeZone;Use NodaTime.DateTimeZone via DateTimeZoneProviders.Tzdb["Europe/Warsaw"]. The TimeZone class is obsolete, does not support historical DST rules, and cannot correctly represent Warsaw's post-2004 CEST transition dates.
P:System.TimeZoneInfo.Local;Use NodaTime.DateTimeZoneProviders.Tzdb["Europe/Warsaw"] with an explicit IANA zone ID. TimeZoneInfo.Local resolves to the server's OS-configured timezone - in UTC containers this is UTC, silently breaking all Warsaw CET/CEST logic.
M:System.TimeZoneInfo.FindSystemTimeZoneById(System.String);Use NodaTime.DateTimeZoneProviders.Tzdb[ianaId] instead. FindSystemTimeZoneById("Central European Standard Time") works only on Windows; Linux requires "Europe/Warsaw". The same binary behaves differently per OS - NodaTime.Tzdb always uses IANA names cross-platform.
M:System.TimeZoneInfo.ConvertTime(System.DateTime,System.TimeZoneInfo);Use NodaTime: localDateTime.InZoneStrictly(zone) or .InZoneLeniently(zone). ConvertTime silently returns a guessed result for ambiguous times (October fold) and may throw for skipped times (March gap) without indicating which case occurred. NodaTime forces explicit handling via AmbiguousTimeException / SkippedTimeException.
M:System.TimeZoneInfo.ConvertTimeFromUtc(System.DateTime,System.TimeZoneInfo);Use NodaTime: Instant.FromDateTimeUtc(dt).InZone(DateTimeZoneProviders.Tzdb["Europe/Warsaw"]). ConvertTimeFromUtc returns a DateTime with Kind=Local tied to whatever the local system zone is, not necessarily Warsaw.
M:System.TimeZoneInfo.ConvertTimeToUtc(System.DateTime,System.TimeZoneInfo);Use NodaTime: localDateTime.InZoneStrictly(zone).ToInstant().ToDateTimeUtc(). ConvertTimeToUtc silently picks standard-time offset for ambiguous local times during October fold, instead of surfacing the ambiguity.
M:System.DateTimeOffset.op_Implicit(System.DateTime);Do not implicitly cast DateTime to DateTimeOffset. The resulting offset depends on DateTime.Kind: Zero for Utc, server-local offset for Local/Unspecified. A Warsaw CEST DateTime (Unspecified) cast on a UTC host gets offset +00:00 instead of +02:00 - a 2-hour silent data corruption. Use NodaTime for explicit zone-aware conversion.
